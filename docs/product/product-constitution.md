# Product Constitution

The Product Constitution defines the principles that should rarely change.

## 1. Trust is the product

Karri does not compete only on price or convenience.

Karri competes on confidence.

## 2. Simplicity wins

The product should feel easy enough for a first-time sender and structured enough for a serious platform.

## 3. Custody must be visible

A package should never feel lost inside the system.

Every important custody change must be recorded and shown clearly.

## 4. Business operations belong on the server

The mobile app should not make final business decisions.

Bookings, custody changes, notifications, trust updates, and reviews must go through controlled backend logic.

## 5. Events power the platform

Important actions emit events.

Events support notifications, analytics, operations, audit history, and future AI.

## 6. Configuration beats hardcoding

Countries, corridors, categories, prohibited items, pricing guidance, and feature availability should be remotely configurable.

## 7. Documentation comes first

No meaningful feature begins with code.

Every meaningful feature starts with a documented problem, user story, acceptance criteria, architecture, security notes, tests, and analytics.
