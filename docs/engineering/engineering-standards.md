# Engineering Standards

Karri engineering should favor clarity, safety, and maintainability.

## Core standards

- Readability over cleverness
- Simple architecture before advanced architecture
- Managed services before custom infrastructure
- Documentation before implementation
- Tests for meaningful business logic
- Security review for trust-sensitive features

## Feature checklist

Every meaningful feature needs:

- Problem statement
- User story
- Acceptance criteria
- Architecture notes
- Security notes
- Analytics events
- Tests
- Documentation update

## Code style

Code should be easy for another engineer to understand quickly.

Important functions should explain why they exist, not just what they do.

## Review standard

A change is not complete until it is:

- Documented
- Implemented
- Tested
- Reviewed against trust, simplicity, transparency, reliability, and delight
